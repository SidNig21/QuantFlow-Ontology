## Objective

Prove the collaboration foundation with **four Hermes term tiles** on one canvas:

1. **Orchestrator** — issues one task  
2. **Worker A** and **Worker B** — receive that task **simultaneously** and produce work  
3. **Reviewer** — receives **both** workers’ submissions, reviews, and **talks back to the orchestrator** with both results cleanly attributed  

All routing is **Kernel-mediated** (host adapter OK). No guest-to-guest side channel. Datasets stay parked.

## Why this order exists (founder)

One Hermes TUI (WO-008d) is a seat. The product needs Hermes as an **orchestrator of other agents on the canvas**. A two-tile ping is too thin. The bar is fan-out to two workers in parallel, fan-in to a reviewer, and a clean talk-back to the orchestrator — the smallest shape that looks like real multi-agent work.

**Defects (must fix before PASS):**

**D1 — Self-logging smoke is not delivery proof.**  
`a2a-4tile-smoke.ts` appends to `delivered/*.log` in the same publish path it later asserts. `capturePath` is opened and never read. Checks (`workerAGot`, `falsifySilent`, …) observe the harness diary, not Hermes TUI / PTY bytes. `proof.json` timestamps even show submissions *before* fan-out wall-clock — the oracle is not trustworthy.  
**Fix:** Assert bytes that crossed a delivery boundary the publisher does not also write as the oracle (PTY capture read-back, or Kernel artifact + independent observer). Delete or actually use `capturePath`.

**D2 — Dual protocol stacks already drift.**  
Electron `a2a-bus.ts` and headless smoke reimplement envelope/publish/delivery/falsify with different artifact filename shapes.  
**Fix:** One shared envelope/publish core; thin adapters at edges only.

**D3 — `pty.ts` spaghetti + leak (already >1k, grew further).**  
`sessionViewers` never cleared on kill; `sendToSender` falls back to **broadcast all webContents** when viewer missing — global PTY behavior change for A2A.  
**Fix:** Fail closed / scoped display; delete viewer on session teardown; do not mutate global send semantics for the demo.

**D4 — Global singleton bus + canned movie as product IPC.**  
Process-wide `seats` + `deliveryEnabled`; `qf:a2a:runProof` hardcodes worker bodies (`alpha-ready` / `beta-ready`) — host impersonates every hop. Order allows a harness, not selling the choreography as the bus surface.  
**Fix:** Instance/context-scoped bus; IPC = spawn seats + publishAndDeliver (or equivalent); proof choreography stays harness-only. Explicit delivery channel (`display` | `stdin` | `both`), default display-only.

**D5 — “Simultaneous” asserted by ISO string equality.**  
Same synchronous loop → same timestamp is tautological. Assert one `dispatch_id` + multi-target delivery instead.
