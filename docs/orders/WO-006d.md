> **Builder: this file plus the files it names are your entire context.** This is a restyle-only order: zero behavior changes, zero layout rewrites, zero React refactors. If a change would alter what a window *does*, stop.

## Objective

One token stylesheet that every window imports, all hardcoded colors and fonts migrated onto it, and a falsifiable gate that makes off-palette styling impossible — so the settings page, terminal, canvas, and every future surface are provably one build.

## Context pack

- `docs/BLUEPRINT.md` Laws (for tone; no law changes here)
- `collab-electron/src/windows/` — the 12 per-window CSS files (measured 2026-07-19: `session-tile/src/styles.css`, `agent-chat/src/styles.css`, `shell/src/shell.css`, `tile-list/src/App.css`, `terminal-list/src/App.css`, and the rest; `shared/` currently contains **no** shared styles)
- Import idiom (measured): each window's entry does `import "./styles.css"` in `src/main.tsx` (or equivalent)

## The token set (founder-sourced; use these values verbatim — invent nothing)

```css
/* ground & surfaces */        /* accent & spectrum */          /* semantic */
--qf-ink:      #07090C;        --qf-accent: #B7FF00;            --qf-ok:   #4FC98B;
--qf-surface:  #0B0F14;        --qf-teal:   #2FE6CF;            --qf-warn: #E0A63C;
--qf-raised:   #111721;        --qf-violet: #C79BFF;            --qf-fail: #E5604C;
--qf-line:     #1D2530;
--qf-line-soft:#141B24;        /* text */
                               --qf-text:  #E7ECF2;
/* rhythm */                   --qf-muted: #69758A;
--qf-u: 8px;  --qf-r: 10px;    --qf-faint: #3A4252;

--qf-mono: "Geist Mono", ui-monospace, "SF Mono", Menlo, monospace;
--qf-sans: "Geist", -apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif;
```

Rules the primitives must encode: the accent (and glow) appears **only** on live/running state and the single primary action per surface; labels are 10px mono uppercase with .14em tracking; every padding/gap is a multiple of `--qf-u` (halves allowed in dense rows); one radius (`--qf-r`; 999px for chips); 1px `--qf-line` borders everywhere.

## Deliverables

1. **`collab-electron/src/windows/shared/qf-tokens.css`** — exactly the custom properties above, on `:root`, nothing else in the file.
2. **`collab-electron/src/windows/shared/qf-base.css`** — shared primitives styled *only* via the tokens: button (primary/quiet), input/select, chip (+ per-state variants matching the transition-table states), status dot (+ `.live` glow), scrollbar, dialog/panel frame, empty-state text, `.qf-label`. No window-specific selectors.
3. **Every window imports both shared sheets before its own CSS**, using the measured entry idiom. Every window, including settings/nav/viewer/terminal — no exemptions.
4. **Migration:** every raw hex color and `font-family` in `windows/**` (CSS files *and* inline `style=`/styled attributes in `.tsx`) is replaced with the nearest token. Mapping rule: nearest token wins; if a value is genuinely unmappable to any token (not just "slightly different"), **stop and list it in your report** instead of inventing a value — that list is founder input, not your call.
5. **Gate `qa/gates/one-skin.ts`**, wired into `qa/run.ts --all`, cold-safe (no deps beyond Bun): scans `collab-electron/src/windows/**` (`.css`, `.tsx`, `.ts`) and goes **red on any raw hex color or `font-family` declaration outside `shared/qf-tokens.css`**. Vendor/generated files may be allowlisted **inside the gate file, each entry with a one-line justification**; an unexplained allowlist entry is a review defect. **Falsify: add a bait `color:#123456` in any window → red; remove → green; paste both.**
6. **Screenshots in the report** — every window, after migration, at rest. The founder judges the *look*; the gate judges the *coherence*. Both are acceptance.
