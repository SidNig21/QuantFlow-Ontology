**1. One bootstrap source (not three).**
Implement **one** honest ingestion path end-to-end (pick the smallest that yields real rows):
- tennis-data.co.uk archive, **or**

**2. Parquet + content hash.**

**3. Kernel registration + lineage.**

**4. DuckDB read proof.**
A small host/test harness opens the dataset pointer and runs a `SELECT count(*)` (or equivalent) via DuckDB against the Parquet path. Paste row count. **Gate (or package test):** fail if any ingestion path inserts bulk fact rows into `kernel.db` (pointer + metadata only).
