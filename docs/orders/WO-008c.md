# WO-008c — Hermes host-bridged ACP (Outcome A outside the guest)
status: open
assignee: builder
depends: WO-008b (probe held — in-guest native exec impossible on AgentOS 0.2.7)
created: 2026-07-20

## Objective

Hermes speaks ACP as a **real host process** (`hermes acp`), driven by QuantFlow’s Electron host. Dock **Spawn** still admits a session and opens a **session tile** — same founder UX — without asking AgentOS’s WASM-only guest to exec Python/ELF. Handshake only; no live turn (WO-008a).

## Why this order exists (measured)

WO-008b proved: authorized mounts make Hermes files **visible** in the guest; guest `spawn` of Hermes/Python fails with `ERR_NATIVE_BINARY_NOT_SUPPORTED` / WASM refuse. Bundle hits the same wall. See `species/hermes/D0-MOUNT-PROBE.md`. AgentOS remains correct for **Node/WASM ACP guests** (toolloop, critic-mock, future Pi-in-AgentOS). Hermes is a **host-bridged** species.

## Context pack

1. `species/hermes/D0-MOUNT-PROBE.md` + WO-008b verification record
2. `collab-electron/src/main/agent-host.ts` — `admitAndStartSession` / `runTurn` (WO-007b)
3. Hermes programmatic integration — **ACP** = JSON-RPC over stdio (`hermes acp`); same protocol AgentOS uses for in-VM guests

## Deliverables

**0. Host ACP client for Hermes (handshake).**
On the **Electron main / host** side (not inside AgentOS guest exec): spawn `hermes acp` (absolute `HERMES_BIN` from founder config / PATH resolve recorded in the report — paths only, never secrets) as a stdio child; complete ACP initialize + `session/new` (or equivalent); adopt the **ACP session id** into Kernel `create_agent_session` + `start_agent_session`. **Send no prompt.** Paste session id + proof of no chunks/artifacts.

Prefer a small module (e.g. `hermes-host-bridge.ts`) owned by `agent-host` routing — extract-first if `agent-host.ts` would grow past comfort; `renderer.js` / `tile-manager.js` still must not grow.

**1. Species launch routing (generic, not a one-off hack).**
`admitAndStartSession("hermes")` (and dock spawn) must take the **host-bridge** path for Hermes. In-VM AgentOS `createSession` remains for AgentOS packages (toolloop / critic-mock). Document the rule in one place (species flag, package manifest field, or host allowlist of `launch: "host_acp"` — pick the smallest data-driven shape; **stop and report** if you need a schema column). No `if (species === "hermes")` scattered across renderer.

**2. Dock → tile UX.**
Register Hermes → dock lists it → **Spawn** → session tile opens, Kernel `created`+`started`, chip/status live. **Do not Run turn** on Hermes. Paste screenshot + event types. Critic-mock / toolloop must still spawn via AgentOS (regression: one mock admit still green).

**3. d0-smoke Outcome A (host path).**
Update `species/hermes/d0-smoke.ts` (or a sibling host-smoke) so Outcome A means: host-bridged handshake succeeded. Keep typed failure modes honest (UNKNOWN vs old B). Exit 0 on A. No prompt.

**4. Lifecycle hygiene.**
Cancel / close from dock (legal edges) tears down the host child; no orphan `hermes` processes (count before/after). Cold reopen: no phantom `running` Hermes rows (Law D pattern).
