# WO-004 — Runtime ownership proof: one session ID, no second server

> **Builder: this file plus the documents it names are your entire context. Do not improvise beyond it.**
> **This is a PROOF order, not a feature order.** A clean, well-evidenced failure is a successful outcome. See §Fallback.

## Objective

Prove the L2 bet: **AgentOS owns the public session lifecycle → a custom ACP agent bridges → Vercel AI SDK's `ToolLoopAgent` owns the model/tool loop**, with **one session ID** visible end to end and **no second server process**. If the chain cannot hold, say so with evidence — that outcome triggers the named Mastra fallback by a follow-up order.

## Deliverables

1. **`tools/agentos-host/`** — harvested per above, its own tests green.

2. **`tools/runtime-proof/`** — a standalone Bun + TypeScript package containing the proof harness. It may import from `tools/agentos-host`. It is a spike: no Electron, no canvas, no Kernel, no SQLite.

3. **The chain, minimally wired:**
   - an **AgentOS session** started via the host, which is the owner of the public session lifecycle and the source of the session ID;
   - a **custom ACP agent** (`@agentclientprotocol/sdk`, v1.2.1) as the bridge — the ACP layer must carry the *same* session ID it was given, never mint its own;
   - **`ToolLoopAgent`** from the Vercel AI SDK (`ai`, v7.x) with `@ai-sdk/anthropic`, owning the model/tool loop inside that session.

4. **One harmless tool** — a pure function the model can call, e.g. `echo_upper(text: string) -> string`. No filesystem, no network, no side effects.

5. **Incremental streaming** — assistant output must arrive as a stream of chunks, not one final blob. Prove it: record chunk arrival timestamps and assert more than one distinct chunk.

6. **Cancel mid-turn** — cancel a run while the model is streaming; assert the stream stops, the session reaches a terminal state, and **no orphan process or timer remains**.

7. **The proof assertions** (this is the order's whole point — each must be a runnable test, not prose):
   - **P1 · One session ID.** The ID reported by AgentOS, the ID carried by the ACP layer, and the ID attached to the tool-loop turn are **identical**. Assert equality on the actual runtime values, not on a variable you passed around by hand.
   - **P2 · No second server.** Exactly one process listens. Enumerate listening sockets (or the host's own registry) before and after starting a session and assert no additional listener appeared. A second Eve-style server process fails this order.
   - **P3 · Tool call inside the session.** The tool is invoked by the model loop and its result reaches the final assistant message.
   - **P4 · Cancel is clean.** Per deliverable 6.

8. **A repo qa gate** — `qa/run.ts` gains a `runtime-proof` gate that runs the proof suite. It must appear in `--list` and pass in `--all`. If the suite requires an API key, the gate must **skip with a clear message** when the key is absent rather than fail — CI has no key.

## Contract

- Bun + TypeScript strict.
- **Model:** `claude-opus-4-8` via `@ai-sdk/anthropic`, model ID read from an env var with that as the default. Three constraints on this model — violating any returns HTTP 400: **do not send `temperature`, `top_p`, or `top_k`**; **do not send `thinking: {type:"enabled", budget_tokens: N}`** (use `{type:"adaptive"}` or omit); **no assistant-turn prefill**. Keep `max_tokens` small (≤ 1024) — this is a plumbing proof, not a capability demo.
- API key from the environment (`ANTHROPIC_API_KEY`). **Never commit a key, never print one, never write one into a test fixture or log line.**
- **Law F holds:** actor-internal states are the runtime's business. Do not model them as ontology types and do not add Kernel objects. This order writes **no** durable state — no SQLite, no JSON state file, no `canvas-state`.
- Do not touch `collab-electron/`, `qf-kernel-schema/`, or any `docs/` file except your report.
- If the ACP layer or `ToolLoopAgent` forces a second session identity or a second server, **stop and report** — do not invent a shim to paper over it. That is the finding.
