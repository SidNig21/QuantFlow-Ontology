# WO-004a — Make the runtime proof assert what is actually true

> **Builder: this file plus `docs/orders/WO-004.md` (including its amended verdict) are your context.**
> This is a **correction order**. The code works; two assertions claim more than they measure.

## Objective

Replace WO-004's overstated assertions with ones that measure real values, so the proof cannot pass on a self-assignment or a lost race.

## Background (read the amended verdict in `WO-004.md` first)

`acp-main.ts` writes `toolLoopSessionId: acpSessionId` — a self-assignment. Nothing is passed into `agent.stream()`, and `ToolLoopAgent` has no session concept (`ai@7.0.31`). P1's third comparison is therefore a value against itself. The order that demanded it was wrong; the fix is to assert what the architecture genuinely provides.

## Deliverables (all inside `tools/runtime-proof/`)

1. **P1 restated — two layers, measured.** Assert exactly this and nothing more:
   - the ID AgentOS reports from `createSession` / `listSessions` **equals** the ID the guest minted in `newSession`, read back across the process boundary;
   - **delete** the `toolLoopSessionId` field and its assertion. Do not replace it with another echo.

2. **P1b — the loop is session-scoped, measured.** Prove the tool loop runs *inside* the session rather than carrying its identity:
   - every `sessionUpdate` the guest emits during the turn carries the session ID, asserted host-side from **received ACP notifications** (not from the receipt);
   - a prompt to an **unknown session ID is rejected**, proving the routing is real and not incidental.

3. **P4 strict.** `stopReason` must be **exactly `cancelled`**. Remove the `["cancelled","end_turn"]` alternation. If the mock can finish before cancel lands, slow it until the race is not close, and assert at least one chunk arrived **before** cancel and none after drain.

4. **P4 orphan-process assertion.** The agent is a **stdio child process**, so a leaked child is the real orphan risk and the current gate only counts listeners. Enumerate descendant processes before and after, and assert zero survive. (Behaviour is already correct — measured during verification — this makes it a gate.)

5. **Kill the `/tmp` receipt as a cross-layer oracle.** `/tmp/qf-runtime-proof-receipt.json` is a fixed path shared by every concurrent run, its write is swallowed by `catch {}`, and it is parsed with a cast. Replace with either ACP-native evidence (preferred — the notifications you already receive) or a per-run unique path that **fails loudly** on write error and is validated, not cast. No silent failure may be able to masquerade as a pass.

6. **One shared fixture.** P1/P1b/P3 each pay a full `AgentOs.create` today. Use one fixture per suite and pack the agent **once** (currently packed in `beforeAll` *and* again in the qa gate).

### Defect D1 — a self-comparison survived the order that existed to kill self-comparisons

`tools/runtime-proof/src/proof.ts:96`:

```ts
const listedSessionId = listed.find((s) => s.sessionId === agentOsSessionId)?.sessionId;
```

This filters *by* the value and returns that same value, so `expect(run.listedSessionId).toBe(run.agentOsSessionId)` (`proof.test.ts:40`) is a tautology — it cannot fail.

**Measured consequence, before ranking it:** this is *not* a coverage hole. The verifier's corruption test failed loudly at the `throw` on line 97. The behaviour is genuinely guarded; the `expect` is decorative. That is materially milder than WO-004's `toolLoopSessionId`, which had no guard at all.

**Required fix — assert the table without filtering on the answer:**

```ts
const listed = os.listSessions();
// assert against the table's contents, not against a lookup keyed by the expected value
expect(listed.map((s) => s.sessionId)).toContain(run.agentOsSessionId);
```

Keep the loud failure for the "not listed at all" case. **Prove the new assertion can fail:** make `listSessions` report a different ID, show P1 red, restore, show green. Paste both outputs.
