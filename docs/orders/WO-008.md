## Measured facts (verifier + reviewer, 2026-07-19 — cite or probe, never guess)

- **`hermes acp` is a Zed-style ACP stdio server** (vendor CLI reference; `hermes acp --check` OK on this machine). ⚠ The other "ACP" (agentcommunicationprotocol.dev, REST, now "A2A") is a **different protocol** — off-surface for this order.
- **Guest env is caller-supplied only:** `agent-os.js:2693` `const launchEnv = { ...options?.env }` — host `PATH`/`HOME` are **not** inherited; sessions run in the AgentOS guest space (overlay filesystem modes `ephemeral|read-only`, `layers.d.ts`). **A shim that assumes host PATH or `~/.hermes` visibility is wrong by construction.**
- **Hermes has two session identities** (reviewer, Hermes source): the ACP session id (`acp_adapter/session.py:204`, `uuid.uuid4()`) and an internal Hermes session id (`provenance.py`). **The Kernel adopts the ACP id only.**
- Hermes ACP boots with `use_unstable_protocol=True` — wire-shape drift vs `@agentclientprotocol/sdk@1.2.1` is possible; deliverable 0 must catch it.
- Hermes emits ACP **permission requests** with deny-on-timeout; the host has **no** permission handler (measured). Builder gates therefore never send a prompt; the founder's live turn waits for **WO-008a** (permission bridge), by design and on the ladder.
- **Data constraint (runtime audit, 2026-07-19): the `agent_definition.name` must equal the package's `agentos-package.json` agent name** — a mismatch fails `createSession(name)`. Both must be `"hermes"` (and `"critic-mock"` for deliverable 4).
- **Unwired permission requests are auto-denied by the SDK with a warning** (`agent-os.d.ts:573-577`) — a stray Hermes permission request degrades, it does not hang. WO-008a still owns the real bridge.
- `register.ts` opens the DB with `openKernel(dbPath)` from `qf-kernel` (Bun SQLite + `attachKernel`) — the audited import shape; relative `package_ref` values resolve from the repo root (`appRoot()`), matching the boot seed's `tools/runtime-proof/packed/...` pattern.
- App DB path (measured `paths.ts`/`kernel.ts`): `~/.collaborator/dev/worktree-<12hex>/kernel.db` — `register.ts` must take the DB path as a **required argument**; the demo doc prints how to find it. No guessed paths.
