## Deliverables

**1. Schema amendment (authorized verbatim; nothing beyond it).**
- `agent_definition` gains `package_ref TEXT NOT NULL` (comment: "AgentOS package this species launches — the plug half of the row"). Table is empty in every real DB; no migration compatibility work.
- New creation action `register_agent_definition` (`defineAction`, experimental): input `name` (unique species name), `role`, `package_ref`, optional `system_prompt_ref`. **Object identity: `id = name`** — the registry's natural key; a duplicate registration is therefore an existing-row hit and gets the typed rejection (same shape as `publishArtifact`'s existing check). Receipts and demo queries stay stable across implementations. §Actions in `ONTOLOGY_SCHEMA.md` gains that one token, placed with the other creation actions.
- `creationCommands` entry `{ action: "register_agent_definition", object_type: "agent_definition", event: "agent_definition.registered" }` + handler in the D3 dispatch table: reject a duplicate `name` with a typed error (registry names are identities); one row, one event, one transaction.
- Regenerate goldens; conformance expectations update only as forced. `doc-action-surface` falsified one-sided (doc-only bait → red → revert → green); paste both.

**2. Boot seed through the front door.** At app-ready, if no `agent_definition` row named `qf-toolloop` exists, the main process registers it **via `execute("register_agent_definition", …)`** — never a direct insert. Idempotent across relaunches (row count stays 1; paste proof).

**3. The host resolves species from rows, not code.** `agent-host.ts`'s in-code species map shrinks to nothing or to pure path-resolution plumbing: spawn takes a species *name*, loads the row, launches `package_ref`. **SDK path (measured, `@rivet-dev/agentos-core` `agent-os.d.ts:554`):** packages for rows known at host creation go into `AgentOs.create`'s `software` list; a row registered **after** the host exists is admitted via `await os.linkSoftware(...)` before `createSession(name)` — no host re-create. `package_ref` is a path resolved from the app root (absolute allowed); resolution failure is the typed error. A registered row whose `package_ref` doesn't resolve fails the spawn loudly with a typed error (never a silent fallback).

**4. The dock rail** in the shell window, skinned entirely by existing tokens/primitives (one-skin stays green):
- **Species section:** every `agent_definition` row via a new read-only `qf:definitions:list` IPC — name, role, spawn affordance. **No species name may appear as a string literal in any renderer source** — that is the contract's "no hardcoded species list," and the gate below enforces its consequence.
- **Sessions section:** live `agent_session` rows with state chips per the transition table; **controls follow legal edges only (measured `transitions.ts`): Cancel appears on `running`/`blocked` — a `starting` row shows its chip and no control** (it resolves to `running` or `failed` on its own); Close on `cancelled`/`failed`; wired to the existing 006c channels.
- Refresh on a pushed invalidation event (reuse the 006c push idiom) — no polling loops.

**5. Law D for the dock.** Force-kill during a live session → relaunch → the dock rebuilds species and sessions from `kernel.db` alone; the interrupted session shows a terminal state (006c reconciliation), never phantom `running`. Part of the demo script.

**6. The cube empty-state.** Integrate `flow-cube-watermark.js` on the shell canvas per its own usage header (`getTileCount` drives bold↔faded). Its internal spectrum constants are the founder's palette and may stay verbatim; do not restyle the engine.
