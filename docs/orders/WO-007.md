## Deliverables

**1. Schema amendment (authorized verbatim; nothing beyond it).**
- `agent_definition` gains `package_ref TEXT NOT NULL` (comment: "AgentOS package this species launches — the plug half of the row"). Table is empty in every real DB; no migration compatibility work.
- New creation action `register_agent_definition` (`defineAction`, experimental): input `name` (unique species name), `role`, `package_ref`, optional `system_prompt_ref`. §Actions in `ONTOLOGY_SCHEMA.md` gains that one token, placed with the other creation actions.
- `creationCommands` entry `{ action: "register_agent_definition", object_type: "agent_definition", event: "agent_definition.registered" }` + handler in the D3 dispatch table: reject a duplicate `name` with a typed error (registry names are identities); one row, one event, one transaction.
- Regenerate goldens; conformance expectations update only as forced. `doc-action-surface` falsified one-sided (doc-only bait → red → revert → green); paste both.

**2. Boot seed through the front door.** At app-ready, if no `agent_definition` row named `qf-toolloop` exists, the main process registers it **via `execute("register_agent_definition", …)`** — never a direct insert. Idempotent across relaunches (row count stays 1; paste proof).

**3. The host resolves species from rows, not code.** `agent-host.ts`'s in-code species map shrinks to nothing or to pure path-resolution plumbing: spawn takes a species *name*, loads the row, launches `package_ref`. A registered row whose `package_ref` doesn't resolve fails the spawn loudly with a typed error (never a silent fallback).

**4. The dock rail** in the shell window, skinned entirely by existing tokens/primitives (one-skin stays green):
- **Species section:** every `agent_definition` row via a new read-only `qf:definitions:list` IPC — name, role, spawn affordance. **No species name may appear as a string literal in any renderer source** — that is the contract's "no hardcoded species list," and the gate below enforces its consequence.
- **Sessions section:** live `agent_session` rows with state chips per the transition table; Cancel on `starting/running/blocked`, Close on `cancelled/failed`; wired to the existing 006c channels.
- Refresh on a pushed invalidation event (reuse the 006c push idiom) — no polling loops.

**5. Law D for the dock.** Force-kill during a live session → relaunch → the dock rebuilds species and sessions from `kernel.db` alone; the interrupted session shows a terminal state (006c reconciliation), never phantom `running`. Part of the demo script.

**6. The cube empty-state.** Integrate `flow-cube-watermark.js` on the shell canvas per its own usage header (`getTileCount` drives bold↔faded). Its internal spectrum constants are the founder's palette and may stay verbatim; do not restyle the engine.

**7. Debts #15/#16 land.** `one-skin` scans `.js` and `rgb()/rgba()/hsl()` syntax; shell's legacy canvas palette + `shell.css` remnants migrate to tokens. The flow-cube engine files are the gate's **only** allowlisted exception (one line each, justified: founder-authored brand engine, palette is the token source, not a divergence). Falsify the widened gate: bait `rgb(1,2,3)` in a window `.js` → red → remove → green; paste both.

**8. Headless gate `qa/gates/dock-registry.ts`** — cold-safe with its own dependency closure (the agent-path pattern; a missing dep fails the gate, never the runner). Using the same registry/list/spawn-resolution functions the app uses (import from a shared module, not a reimplementation):
- register a bait species via `execute` → it appears in the definitions list → **red/green:** point the list at anything but the DB (or skip registration) and the assertion fails; show it.
- corrupt a row's `package_ref` → spawn resolution throws the typed error → restore → green.
- duplicate `name` registration → typed rejection, row count still 1.
Wire into `qa/run.ts --all`.

**9. `docs/demos/dock.md`** — founder script: open app → dock shows ToolLoop from the Kernel → spawn → chip walks the transition table → cancel → force-kill mid-run → relaunch → dock rebuilt, honest states → one receipt query showing `agent_definition.registered` and the session events sharing the ledger.
